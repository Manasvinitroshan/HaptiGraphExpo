# HaptiGraph

**Adaptive haptic-acoustic rendering of mathematical functions for accessible graph comprehension.**

HaptiGraph lets users explore mathematical graphs through vibration and sound — no screen required. As a user traverses a function, the phone vibrates at intensities and rhythms that encode slope, curvature, and key landmarks (peaks, valleys, zero-crossings). An on-device adaptive engine (multi-armed bandit + reward model) personalises the feedback pattern over time based on the user's self-reported comprehension.

## Setup

```bash
# 1. Clone the repo
git clone https://github.com/Manasvinitroshan/HaptiGraphExpo.git
cd HaptiGraphExpo

# 2. Install dependencies
npm install

# 3. Start the dev server
npx expo start
```

Scan the QR code with **Expo Go** on your phone (iOS or Android).

To target a specific platform:

```bash
npx expo start --ios      # iOS simulator
npx expo start --android  # Android emulator
```

If port 8081 is busy:

```bash
npx expo start --port 8082
```

## How It Works

### 1. Equation Parsing
`graphParser.js` implements a full recursive-descent parser — no `eval`. The equation string is tokenised, parsed to an AST, and compiled to a reusable evaluator function. 201 points are sampled over x ∈ [−10, 10].

### 2. Numerical Analysis
For each sample point, first and second derivatives are computed via central differences (O(h²) error). These drive the haptic mapping:

| Signal | Source | Encoding |
|--------|--------|----------|
| Vibration intensity | Slope \|f'(x)\| | Light / Medium / Heavy impact |
| Pulse speed | Curvature \|f''(x)\| | 160 / 90 / 35 ms delay |
| Feature event | Landmark type | Distinct notification pattern |

### 3. Adaptive Engine
An ε-greedy multi-armed bandit selects from haptic intensity presets. After each session the user reports their equation guess and confidence; the reward model converts this to a scalar reward and updates the bandit arm weights.

### 4. Audio
Sound feedback is generated by a hidden WebView running the Web Audio API (`AudioEngine.js`). Tone frequency tracks the normalised y position of each point; tone waveform and volume encode slope intensity.

## Running on a Physical Device

1. Install **Expo Go** from the App Store / Google Play.
2. Run `npx expo start` in the project directory.
3. Scan the QR code. Haptic feedback requires a physical device — simulators do not vibrate.

---

## Notes

- Sound playback uses `react-native-webview` to host a Web Audio API context. On first load there may be a 1–2 s warm-up delay.
- The adaptive engine state persists in React component state only — it resets on app restart.
